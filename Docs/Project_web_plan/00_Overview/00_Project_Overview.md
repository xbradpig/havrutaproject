## 1. 프로젝트 배경 | Project Background

### 1.1 시대적 맥락 (ASI Era Context)

우리는 AGI(Artificial General Intelligence)를 넘어 ASI(Artificial Super Intelligence) 시대로 진입하고 있습니다. 이 시대의 핵심 위기는 **기술의 한계가 아니라 인간 정체성의 실종**입니다.

> "어느새 사람들은 '나는 누구인가'를 묻지 않게 되었다.
> '나는 무엇을 할 수 있는가'만이 남았다.
> 존재는 사라지고, 성능만 남았다."
> — Philosophy.md

### 1.2 Havruta Project의 응답

**Havruta Project**는 이 시대적 도전에 대한 기독교적 응답입니다.

- **핵심 정의**: ASI 시대에도 교회와 성도가 정체성을 잃지 않고, 말씀 안에서 서로를 지키며 부흥을 지속시키는 프로젝트 생태계
- **브랜딩**: `Havruta Project — powered by AreteVision / A project by AreteVision`

## 3. 핵심 철학 요약 | Core Philosophy Summary

### 3.1 Havruta의 정의

> **하브루타는 말씀을 함께 읽고 → 외우고 → 질문하고 → 대화하고 → 순종하며 → 서로를 지키는 길이다.**

### 3.2 부흥 루프 (Revival Loop)

```
┌─────────────────────────────────────────────────────┐
│                    부흥 루프                          │
│                                                      │
│   말씀(입력) → 암송(내면화) → 질문(분별)              │
│       ↑                           ↓                  │
│   다시 말씀 ←── 보호(서로 지킴) ← 대화(공동체)        │
│                       ↑           ↓                  │
│                   순종(적용) ←────┘                  │
│                                                      │
└─────────────────────────────────────────────────────┘
```

### 3.3 핵심 대조

| 세상의 방식 | Havruta의 방식 |
|-------------|----------------|
| 더 큰 지능 | 더 깊은 사람 |
| 기능으로 인간 정의 | 창조의 선언으로 인간 보호 |
| 개인 분리 | 가족(주 안의 가족) 연결 |
| 성능 경쟁 | 정체성 보존 |

## 6. 기술 방향 | Technical Direction

### 6.1 권장 기술 스택

| 영역 | 기술 | 근거 |
|------|------|------|
| Framework | Next.js 14+ | SSG/SSR 혼합, SEO 최적화 |
| Styling | Tailwind CSS | 일관된 디자인 시스템 |
| Animation | Framer Motion | 부드러운 인터랙션 |
| CMS | MDX | 콘텐츠 관리 용이 |
| Hosting | Vercel | 글로벌 CDN, 간편 배포 |
| Analytics | Plausible/GA4 | 개인정보 보호 우선 |

### 6.2 성능 목표

- Lighthouse Score: 90+
- First Contentful Paint: < 1.5s
- Core Web Vitals: 모두 Green
